## 快速开始

### 方案一: GitHub Actions 部署 (推荐, 零成本)

1. Fork 或推送到你的 GitHub 仓库
2. 在仓库 `Settings → Secrets → Actions` 中添加通知 Token
3. `.github/workflows/scan.yml` 会自动在每个交易日 14:45 (北京时间) 运行
4. 也可在 Actions 页面手动触发 (`workflow_dispatch`)

### 方案二: Docker 本地部署

```bash
# 1. 复制配置文件
cp configs/config.example.env configs/.env
# 2. 编辑 .env, 填入通知 Token
vim configs/.env
# 3. 构建镜像
docker build -t rally-scanner .
# 4. 运行扫描
docker run --rm --env-file configs/.env rally-scanner
```

### 方案三: 本地直接运行

```bash
# 1. 安装依赖
pip install -r requirements.txt
# 2. 配置环境变量
cp configs/config.example.env configs/.env
# 3. 运行
cd src && python main.py
```

## 策略说明

### 四维硬核动量模型 v2.0

本系统的核心是 **四维硬核动量串联检测模型**，四个指标全部通过才输出信号，任一失败即淘汰并记录原因：

| 维度 | 指标 | 信号条件 | 权重 |
|------|------|----------|------|
| 1. 均线完美多头排列 | MA(5/10/20/60/120) | Price > MA5 > MA10 > MA20 > MA60 且 MA20 > MA60 > MA120 | 30分 |
| 2. 动量爆发与放量破局 | 新高 + 量比 | 创20日或60日新高 + 量比5d>2.0x 且 量比20d>2.0x | 25分 |
| 3. VCP波动率收敛后突破 | 振幅 + K线 + 布林带 | 10日振幅收敛(比>1.3) + 大阳线(>3%) + 布林扩张(>1.5x) | 30分 |
| 4. 大盘环境过滤器 | 指数MA20 | 指数>MA20做多(15分) / <MA20>MA60谨慎(8分) / <MA20<MA60熔断(0分) | 15分 |

**VCP (Volatility Contraction Pattern) 形态量化**:
- Mark Minervini 经典形态: 价格波动率从左到右逐渐收敛 (T1 > T2 > T3)
- 量化方法: 将10日回看期分前后两半, 前半段均振幅 / 后半段均振幅 > 1.3 → 收敛
- 突破确认: 今日收盘 > 前N日最高 + 实体涨幅 > 3% + 布林带宽 > 20日均值 × 1.5

**空仓熔断机制**:
- 当大盘指数跌破 MA20 且跌破 MA60 时, 触发空仓熔断
- 系统进入防御模式, 即使个股四维全部通过也会被过滤
- 推送 "建议全部空仓观望" 警告

**信号等级**:
- **S级**: 四维全通 + 大盘做多环境 (强烈主升浪加速)
- **A级**: 四维全通 + 大盘谨慎环境 (主升浪确认, 但需注意系统性风险)
- **B级**: 部分通过 (候选关注)

**详尽日志**:
- 每只股票的每条规则 PASS/FAIL 都有明确的数值记录
- 被淘汰的股票会记录具体淘汰原因 (如 "MA排列不完美" / "量比5d=1.2x<2.0x" / "VCP振幅未收敛" 等)
- 命中的股票会打印四维全部 PASS 的详细指标数值

> ⚠️ 以上为量化信号扫描结果, 不构成投资建议。
