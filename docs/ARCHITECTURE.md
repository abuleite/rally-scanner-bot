## 核心模块

| 模块 | 文件 | 职责 |
|------|------|------|
| 数据抓取 | `src/data_etl.py` | 涨停股获取(同花顺/东财/腾讯/Baostock) + K线/指数多源故障切换 |
| 动量扫描 | `src/scanner.py` | 四维硬核动量检测 (MA多头/放量突破/VCP/大盘过滤) |
| 因子评分 | `src/factors.py` | 六因子评分模型 (行业/市值/趋势/动量/质量/仓位) + Playbook |
| 报告生成 | `src/report.py` | HTML/Excel/Markdown 三格式报告 |
| 通知推送 | `src/notifier.py` | 多渠道 Webhook 推送 (PushPlus/Telegram/钉钉/企微/Discord) |
| 流水线 | `src/main.py` | 四段式流水线编排 |
| 运维工具 | `src/utils.py` | 交易日历/健康检查/磁盘清理/异常兜底 |

## 双模型交叉验证

系统采用"四维动量 + 六因子"双模型交叉验证:

- **动量扫描**: 识别技术面突破 (放量+均线+VCP), 重视入场时机
- **因子评分**: 评估基本面适配 (行业+市值+趋势), 重视选股逻辑
- **交叉验证**: 高动量+高因子评分 = 高置信度信号

## 零成本部署

### 方案一: GitHub Actions (推荐)

```bash
# 1. Fork 或推送到 GitHub 仓库
# 2. 在 Settings → Secrets → Actions 添加:
#    PUSHPLUS_TOKEN (必填, 微信推送)
# 3. 自动每个交易日 20:00 (北京时间) 运行
```

### 方案二: Docker 部署

```bash
cp configs/config.example.env configs/.env
# 编辑 configs/.env 填入 Token
docker-compose up -d
```

## 依赖清单

- **数据源**: baostock / akshare / efinance / yfinance / requests
- **技术指标**: ta (纯Pandas实现, 零编译依赖)
- **报告输出**: openpyxl (Excel) / 纯 SVG 雷达图 (零额外依赖)
- **通知推送**: requests (纯HTTP调用)

## 评分参数调优

| 参数 | 默认值 | 说明 |
|------|--------|------|
| VOLUME_SURGE_RATIO | 2.0 | 放量倍数阈值 |
| VCP_CONTRACTION_THRESHOLD | 1.3 | 振幅收敛比 |
| BOLLINGER_EXPLODE_RATIO | 1.5 | 布林带扩张倍数 |
| CANDLE_BODY_MIN_PCT | 3.0 | 大阳线最小实体% |
| MIN_CHANGE_PCT | 5.0 | 涨幅过滤阈值 |
| SCAN_ZT_ONLY | 1 | 涨停股扫描模式 |
